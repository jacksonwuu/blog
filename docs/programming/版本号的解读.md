# 版本号的解读

很多软件都会带有例如v1.4.5.3这样的版本号，我非常好奇这些版本号都代表了什么意思，于是见过我的查阅，我找到了一份接受度最高的版本号解读：

w.x.y.z

* w - 主要版本发布，包含很多新特性。
* x - 新特性发布，但是没有重大特性更新。
* y - Bug 修复发布。
* z - 补丁发布（很少用到）。

例子：

* 0.9     - 第一个 Beta 版本
* 0.9.1   - 第二个 Beta 版本
* 0.9.1.1 - 第二个 Beta 版本的 Bug 修复
* 1.0     - 第一个正式版本
* 1.1     - 添加一个新特性
* 1.1.1   - Bug 修复
* 2.0     - 重大发布

前一个数字 +1 的话，后面的数字全部归零。
